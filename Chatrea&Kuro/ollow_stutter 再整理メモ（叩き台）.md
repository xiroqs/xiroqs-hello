# follow_stutter 再整理メモ（叩き台）

## まず固定したいこと

ここまでの実機感触から、**固定位置のパパパパ** が By This River 的な質感に最も近かった。  
したがって、follow_stutter の核は

- 固定位置の局所断片
- `metro -> a_time -> sel 0 -> startloop`
- 同じ頭を何度も踏み直す粒感

にあると考えている。

ただし、**固定位置そのものをゴールにしたいわけではない**。  
最終的には、**その粒感を保ったままメロディラインに沿って進行することがマスト**。

---

## 現在の認識

### 1. fixed-position retrigger は正解に近い
- line~ で後段を切る方向より、
- 固定位置を `startloop` で再侵入する方向の方が近かった

### 2. メロディライン追従は必須
- ただパパパパしているだけではなく、
- **メロディラインに沿って進行すること** が必要
- つまり「固定位置の気持ち良さ」は核だが、最終形ではない

### 3. a_time は時間制度の中核
- a_time は単なる counter/%/sel の代用品ではなく、
- **演奏に別の時間制度を与える abstraction**
- trigger 側は機械的でよい

### 4. ユーザー操作は ms 基準
操作値は比率ではなく ms で持ちたい。

- `window_ms`
- `step_ms`
- 必要なら `pulse_ms`

wav 長は内部参照用に留めたい。

## いまズレたくない点

### NG寄り
- fixed-position の気持ち良さを壊してしまうこと
- line~ や gate 的な方向を主役にしてしまうこと
- wav 長比率をユーザー操作の前面に出すこと
- 毎回押す latch ボタン中心の設計

### OK寄り
- fixed-position retrigger を核にする
- trigger は `metro 100 -> a_time -> sel 0`
- その上で **メロディラインに沿う追従** を最小追加で考える
- reset ボタンは状態初期化用として有り

## 今回お願いしたいこと

### 主題
**固定位置のパパパパの気持ち良さを壊さずに、どうやってメロディラインへ沿わせるか**  
を考えたい。

### 欲しい検討対象
1. fixed-position retrigger を核としたまま、
   **最小限の追従** をどう足すか
2. その追従が、
   - 単なる step 進行ではなく
   - **メロディラインに沿っている** と言えるには何が必要か
3. `window_ms` / `step_ms` を ms 基準で保ったまま、
   どこまで追従設計できるか
