# follow_stutter 再整理メモ（修正版）

## 前提
今回やりたいことは、少なくとも以下の 2系統がある。

---

## 1. scan-capture 型（本命）
fixed-position retrigger ではなく、  
**連続で進んでいる wav / playhead を tick ごとにスキャンして、その瞬間の位置を freeze / capture する構造** を考えたい。

つまり、

- 裏では wav は連続進行
- 表では `a_time` の tick ごとに現在位置を freeze / capture
- その結果として、パパパパしながらメロディラインに沿う

この理解で設計を組み直したい。

### 補足
ここでは、固定位置を何度も踏むのではなく、  
**進んでいる source の「その瞬間」を tick ごとに取る** のが主旨。

---

## 2. waveform-loop 型（Switchで切り替える別モード）
Switch 機能で、`waveform~` で選択した部分を読ませて、  
**その範囲を loop 化するモード** も欲しい。

つまり、

- scan-capture 型とは別に
- waveform で選択した範囲を材料にする loop モードを持ちたい

---

## 3. reverse
上記 2 の waveform-loop 型では、**reverse も欲しい**。  
イメージとしては V2 のような方向。

ここでの reverse は、
wav 全体の進行方向ではなく、  
**loop 化した範囲の中で reverse する** 方向を考えている。

---

## 4. ms 可変
どちらのモードでも、長さに関する操作値は **ms 可変** にしたい。

想定しているのは少なくとも、

- `window_ms`
- `step_ms`
- 必要なら他の時間長

で、比率ではなく ms で直接触りたい。

---

## 5. reset について（重要）
ここで重要なのは、単なる「waveform の見た目の reset」ではなく、  
**そのモードで使っている参照基準を reset できること**。

### A. scan-capture 型の reset
- 連続進行している参照位置 / current 的な基準を戻す
- 次の tick から、もう一度その基準で capture し直せるようにする

### B. waveform-loop 型の reset
- waveform の selection / loop 範囲の基準を戻す
- 前の選択範囲や reverse 状態が残りっぱなしにならないようにする

つまり、  
**reset は 1個でもよいが、mode ごとに意味が違う** と考えている。

### 一文で言うと
**waveform の reset が重要というより、  
その時使っている参照基準を reset できることが重要**。

---

## 6. 今回お願いしたいこと
以下の前提で、概念図と実装図を分けて再提案してほしい。

1. 本命は scan-capture 型
2. Switch で waveform-loop 型へ切り替えたい
3. waveform-loop 型では reverse も扱いたい
4. どちらも ms 可変
5. reset は mode ごとに意味が違う前提で設計したい

---

## 一文要約
**本命は、連続進行する wav/playhead を tick ごとに freeze / capture する scan-capture 型。  
それとは別に、Switch で waveform 選択範囲を loop 化するモードも持ちたい。  
後者では reverse も扱いたく、どちらも ms 可変で操作したい。  
また reset は、waveform の見た目ではなく、そのモードで使っている参照基準を戻すものとして設計したい。**
