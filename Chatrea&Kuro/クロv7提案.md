## 0. クロの理解

これまでの v3〜v6 はすべて「**鳴らす位置を自前で決める**」発想だった。
固定位置を再発火する (v3) / current_start を進める (v4, v6) / 1周単位で進める (v5)。
どれも **位置の制御を自前で持つ** 構造。

チャトレアの scan-capture 型はそれとは根本的に違う:

|旧 (v3〜v6)         |新 (v7 / scan-capture)   |
|------------------|------------------------|
|鳴らす位置を自前管理        |wav 自身が連続進行             |
|current_start を進める|tick の瞬間に「今どこか」を取る      |
|折り返しロジック必要        |折り返しは scanner の loop が自動|
|メロディに沿う ≠ 自然成立    |メロディに沿う = **自動で成立**     |

「進んでいる source の『その瞬間』を tick ごとに取る」という言葉から、
playhead 側が主役で、tick はスナップショットのタイミングを決めるだけ、と読み取った。

## 1. 概念図

### scan-capture 型 (mode 1: 本命)

```
┌────────────────────────────────────────┐
│ scanner (裏で連続進行する playhead)      │
│                                         │
│ groove~ source loop 1                   │
│   → sync (0.-1. の進行位置)              │
│   → 音は出さない (*~ 0)                  │
└────────────────────────────────────────┘
            │
            │ sync signal (playhead 位置)
            │
            ↓
   ┌─────────────────────────┐
   │ tick の瞬間に snapshot    │
   │                         │
   │ a_time → sel 0          │
   │      → snapshot~        │
   │      → 現在位置 ms        │
   └─────────────────────────┘
            │
            ↓
┌────────────────────────────────────────┐
│ capturer (短い freeze を発音する)        │
│                                         │
│ loop_min = capture 位置                  │
│ loop_max = capture 位置 + window_ms      │
│ groove~ source startloop                │
│   → 音が出る                             │
└────────────────────────────────────────┘
            │
            ↓
          ezdac~
```

### 役割分担

|layer          |役割                               |音   |
|---------------|---------------------------------|----|
|scanner        |連続進行する playhead を提供              |出さない|
|tick + snapshot|playhead の現在位置を tick の瞬間に capture|-   |
|capturer       |capture した位置から短い window を鳴らす     |出す  |

### waveform-loop 型 (mode 2)

scanner を使わず、capturer が直接 waveform 選択範囲を loop:

```
[r selstart_v7] → cap_min
[r selend_v7]   → cap_max
                  ↓
              capturer (groove~)
                  ↓
              ezdac~

speed (sig~) は flonum で可変:
   1.  = 順進行
  -1.  = reverse
```

mode 2 は v2 系 follow に近い構造。

## 2. mode 切替の構造

```
[mode toggle]  (0 = scan-capture / 1 = waveform-loop)
       │
       ↓
    [switch 2]
     ↑   ↑
     │   │
     │   └ (mode 2 のとき) selstart/selend
     │
     └ (mode 1 のとき) snapshot からの値
       ↓
    capturer の loop_min / loop_max へ
```

`switch` は前回チャトレアが指摘してくれた「2入力 → 1出力で選択」の正しい用途で使える。

-----

## 3. ms 可変の扱い

両モード共通:

- **`window_ms`**: capture / loop の長さ (両モード必須)

mode 1 固有:

- scanner の速度は `sig~ 1.` 固定
- step_ms は **不要**。scanner が連続進行するから、「どれだけ進むか」は scanner の自然進行と tick 間隔で自動的に決まる
- これが scan-capture の利点。tick が遅くても速くても、scanner はメロディを連続進行している

mode 2 固有:

- **`speed`** (1. or -1.): reverse 用
- loop 範囲は selection から取るので、数値入力不要

-----

## 4. reset の mode ごとの意味

### mode 1 (scan-capture) の reset

- **scanner の playhead を loop_min に戻す**
- 方法: scanner の groove~ に `startloop` を送る
- 耳で分かる形: 次の tick で取得される位置が wav の頭 (or selection の頭) からになる

### mode 2 (waveform-loop) の reset

- **capturer の loop 範囲を現在の waveform selection に合わせ直す**
- 方法: selstart / selend を capturer の loop_min / loop_max に再送信 + startloop
- 耳で分かる形: 直前の selection / reverse 状態がクリアされ、今の選択範囲を頭から鳴らし直し

両方とも、共通の reset ボタンから mode に応じて分岐する (`gate 2` で振り分け)。

> **重要**: チャトレアの整理通り、reset は「waveform の見た目」ではなく
> 「**そのモードで使っている参照基準**」を戻すもの。

```
[buffer~ source_v7 30000 2]   ← 共有 buffer

────────────────────────────────────────
① selector UI (v3〜v6 と同じ)
   waveform~ ×2 → send selstart_v7 / selend_v7
────────────────────────────────────────

────────────────────────────────────────
② scanner layer (mode 1 でのみ参照)
────────────────────────────────────────

[sig~ 1] [r scan_min] [r scan_max] [loop 1 (loadbang)] [r scan_reset]
   │          │            │              │                 │
   ↓          ↓            ↓              ↓                 ↓
[groove~ source_v7 2]
 │  │   │
 L  R   sync (0.-1.)
 │  │
 *~ 0  ← 音を消す

scanner の loop 範囲 (mode 1 時):
  scan_min = 0
  scan_max = buffer の長さ (or selstart/selend を使うサブモード可)

                                            ↓
                                         [t b b]

                                          │      │
                                          │      └→ [startloop]

                                          ↓

                                          │
                                          ↓ (0.-1. の float)

                                      │     │
                                      ↓     ↓

                                      │     │
                                      │   [r selstart_v7]

                                      └─→ [switch 2] ←─┘
                                                │
                                                ↓
                                             [t f f f]
                                              右  中  左
                                              │   │   │
                                              │   │   └ [+ window_ms]
                                              │   │       → send cap_max_v7
                                              │   └ send cap_min_v7
                                              └ (debug flonum)

────────────────────────────────────────
④ capturer layer (音を出す)
────────────────────────────────────────

[sig~ speed] [r cap_min_v7] [r cap_max_v7] [r cap_startloop_v7] [loop 1]
    │              │              │                │              │
    ↓              ↓              ↓                ↓              ↓
[groove~ source_v7 2]
 │  │
 L  R
 *~ 0.7
    │
    ↓
 ezdac~

speed:
  mode 1: 1. (固定)
  mode 2: 1. or -1. (reverse ボタンで切替)

────────────────────────────────────────
⑤ mode 切替 + reset
────────────────────────────────────────

[mode toggle]   (0 / 1)
     │
     ↓
   各所の switch / gate へ

[reset button]
     │
     ↓
  [gate 2] ← mode toggle で振り分け
   │   │
   │   └ mode 2:
   │       cap_min/max を selstart/end に再送
   │       cap_startloop_v7 に bang
   │
   └ mode 1:
       scan_reset_v7 に bang
       (scanner の groove~ に startloop)
```

## 6. 確認したい点

### Q1. scan-capture の理解は合ってる?

「scanner = 連続進行する playhead、capturer = 瞬間位置を短い window で鳴らす」
この **2 groove~ 構造** で、チャトレアの意図は成立する?

もし「1 個の groove~ でやりたい」という制約があれば別アプローチ (phasor~ + buffer~ で自前 playhead) も考えられる。
クロの推奨は **2 groove~ 方式**。構造がクリーン、モード切替が素直。

### Q2. scanner の音は出す? 消す?

- 案A: **消す** (`*~ 0`): 純粋にスキャン用、聴こえる音は capturer のみ
- 案B: 薄く出す (`*~ 0.2` 等): scanner が裏で進んでるのを耳で確認
- 案C: monitor out で別出力: 検証用に一時的に出す

クロの推奨: **案A**。xiroqs の美学として、聴こえる音は capturer だけに絞る方が筋がいい気がする。

### Q3. scanner の loop 範囲 (mode 1 時)

- 案A: **wav 全体** (scan_min=0, scan_max=buffer 宣言長)
- 案B: waveform 選択範囲 (scan_min=selstart, scan_max=selend)
- 案C: 両方を切替できるサブモード化

クロの推奨: **まず案A** でシンプルに始める。サブモードは後回し。
ただし mode 2 の存在を考えると、選択範囲ベースの scanner は冗長かもしれない。

### Q4. snapshot~ の動作

`snapshot~` は bang で signal を float に変換する標準オブジェクト。
sync signal が 0.-1. の連続値なので、tick の瞬間に値が取れるはず。

ただし要実機確認:

- snapshot~ の戻り値の精度
- bang から float 出力までの遅延

万一遅延があれば `number~` で継続取得して保持する方式に変える。

### Q5. mode 2 は v2 follow 相当でよいか

- waveform 選択範囲を毎 tick startloop
- speed flonum で reverse 可
- これは v2 (`follow_stutter_event_v2_reverse.maxpat`) の構造そのまま

クロの理解で OK?

## 7. v6 までとの差分まとめ

|項目        |v6 まで             |v7                          |
|----------|------------------|----------------------------|
|位置の決定     |自前 (current_start)|scanner 任せ                  |
|folding   |if 文で実装           |不要 (scanner が loop)         |
|step_ms   |必要                |不要 (mode 1)                 |
|groove~ の数|1                 |**2** (scanner + capturer)  |
|メロディ追従    |step 進行で近似        |scanner の連続進行で自動成立          |
|reverse   |経路なし              |mode 2 で speed 可変           |
|reset     |latch_start に戻す   |scanner playhead を startloop|
|mode 切替   |なし                |switch で 2 mode 切替          |

## 一文要約

scan-capture 型は、scanner (連続進行する playhead) と capturer (瞬間位置を短く鳴らす) の
2 groove~ 構造で実装する。
mode 2 (waveform-loop) は v2 follow + reverse と同等で、switch で切替。
reset は mode ごとに参照基準 (scanner playhead / waveform selection) を戻す形にする。
ms 可変は window_ms と speed のみで、step_ms / current_start / folding はすべて廃止する。
